# CertBuddy 개발 우선순위 제안서

## 📊 프로젝트 현황 분석

### ✅ 현재까지 완료된 기능
1. **인증 시스템**
   - 로그인/회원가입 화면 및 로직 완료
   - AuthContext를 통한 상태 관리
   - Mock API로 동작 확인 완료

2. **학습 시스템 (핵심 기능)**
   - 객관식 문제 풀이 화면
   - 정답/오답 즉시 XP 지급
   - 오답 자동 복습 리스트 추가
   - 학습 결과 XP 화면

3. **복습 시스템**
   - 복습 문제 목록 조회
   - 복습 모드 문제 풀이 (5XP 문제당)
   - "보너스 XP" 표시
   - 다 틀리면 복습 필요 화면

4. **홈 화면**
   - 사용자 인사말
   - 총 XP 및 연속 학습일 표시
   - 오늘의 학습/복습 시작 버튼
   - 복습 문제 개수 표시

5. **기본 네비게이션**
   - 하단 탭 네비게이션 (홈, 학습, 친구, 마이페이지)
   - 스택 네비게이션 구조

### ⚠️ 현재 상태
- **프론트엔드**: Mock API로 동작 중, 핵심 기능 완료
- **백엔드**: 기본 구조만 존재, 실제 구현 없음
- **데이터베이스**: 설계 및 구현 없음

## 🎯 개발 우선순위 제안

### Phase 1: 프론트엔드 기능 완성 (즉시 시작 가능) ⭐ 최우선

프론트엔드 우선 개발 전략에 따라, Mock API로 모든 기능을 먼저 완성하는 것이 좋습니다.

#### 1-1. 홈 화면 기능 완성 (우선순위: 🔥 최고)
**예상 소요 시간**: 2-3일

**작업 내용**:
- [ ] 추천 자격증 섹션 구현
  - 사용자 학교/학과 기반 추천 자격증 리스트 표시
  - 자격증 카드 컴포넌트 (이름, 설명, 문제 수 등)
  - 자격증 선택 시 해당 자격증으로 학습 시작
- [ ] 친구 랭킹 Top3 섹션 구현
  - 친구 목록에서 XP 기준 상위 3명 표시
  - 랭킹 아이콘 및 순위 표시
  - 친구 프로필 이미지/아바타

**왜 우선인가?**
- 홈 화면의 핵심 기능으로 사용자 경험에 직접적 영향
- Mock API가 이미 준비되어 있어 즉시 구현 가능
- 학습 시작 플로우를 완성하는 데 필수

#### 1-2. 친구 기능 구현 (우선순위: 🔥 높음)
**예상 소요 시간**: 3-4일

**작업 내용**:
- [ ] FriendsScreen 완성
  - 친구 목록 표시 (이름, 학교, XP, 스트릭)
  - 친구 추가 기능 (이메일/ID 검색)
  - 친구 삭제 기능
- [ ] 친구 랭킹 화면
  - 전체 친구 랭킹 (XP 기준)
  - 내 순위 표시
  - 스트릭 랭킹도 함께 표시

**왜 우선인가?**
- 소셜 기능으로 사용자 참여도 증가에 기여
- Mock API가 이미 준비되어 있음
- 홈 화면의 친구 랭킹과 연계되어 일관성 있는 UX 제공

#### 1-3. 마이페이지 통계 강화 (우선순위: ⭐ 중간)
**예상 소요 시간**: 2-3일

**작업 내용**:
- [ ] 누적 통계 추가
  - 총 학습 문제 수
  - 총 정답률
  - 학습한 자격증 목록
  - 일별/주별/월별 학습 그래프 (차트 라이브러리 사용)
- [ ] 성취 배지 시스템 (선택사항)
  - 연속 학습일 배지
  - 총 XP 달성 배지
  - 자격증별 완료 배지

**왜 우선인가?**
- 사용자 동기부여에 중요한 기능
- Mock 데이터로 충분히 구현 가능
- 프로필 화면의 완성도를 높임

### Phase 2: 백엔드 기반 구축 (프론트엔드 완성 후)

프론트엔드가 Mock API로 완전히 동작하는 것을 확인한 후, 백엔드를 구현합니다.

#### 2-1. 데이터베이스 설계 및 구축 (우선순위: 🔥 최고)
**예상 소요 시간**: 3-5일

**작업 내용**:
- [ ] ERD 작성
- [ ] 테이블 스키마 정의
  - users (사용자)
  - certifications (자격증)
  - questions (문제)
  - question_sessions (학습 세션)
  - review_questions (복습 문제)
  - friends (친구 관계)
  - user_stats (사용자 통계)
- [ ] 샘플 데이터 준비

#### 2-2. 인증 API 구현 (우선순위: 🔥 최고)
**예상 소요 시간**: 2-3일

**작업 내용**:
- [ ] JWT 기반 인증 구현
- [ ] 회원가입 API
- [ ] 로그인 API
- [ ] 토큰 갱신 API

#### 2-3. 학습/복습 API 구현 (우선순위: 🔥 최고)
**예상 소요 시간**: 4-5일

**작업 내용**:
- [ ] 문제 조회 API
- [ ] 학습 세션 생성/완료 API
- [ ] 복습 문제 관리 API
- [ ] XP 계산 및 업데이트 로직

#### 2-4. 친구/랭킹 API 구현 (우선순위: ⭐ 중간)
**예상 소요 시간**: 2-3일

**작업 내용**:
- [ ] 친구 추가/삭제 API
- [ ] 친구 목록 조회 API
- [ ] 랭킹 조회 API

### Phase 3: 고급 기능 (선택사항)

#### 3-1. 푸시 알림 (우선순위: ⭐ 낮음)
**예상 소요 시간**: 3-4일

**작업 내용**:
- [ ] Expo Notifications 설정
- [ ] 일일 학습 알림
- [ ] 복습 알림
- [ ] 친구 활동 알림

**왜 낮은 우선순위인가?**
- 핵심 기능이 아니며, MVP 이후에 추가해도 됨
- 백엔드 인프라가 필요함

#### 3-2. 실전모드 (우선순위: ⭐ 낮음)
**예상 소요 시간**: 4-5일

**작업 내용**:
- [ ] 실전모드 UI 설계
- [ ] 시간 제한 기능
- [ ] 실전모드 결과 화면
- [ ] 실전모드 통계

**왜 낮은 우선순위인가?**
- 기본 학습 모드가 먼저 안정화되어야 함
- 추가적인 UX 설계가 필요함

## 📋 권장 개발 순서 (요약)

### 즉시 시작 (1-2주 내)
1. ✅ **홈 화면 추천 자격증 섹션** - 사용자가 학습을 시작하는 핵심 진입점
2. ✅ **홈 화면 친구 랭킹 Top3** - 소셜 기능의 첫 단계
3. ✅ **친구 기능 전체** - 친구 추가, 목록, 랭킹

### 단기 (2-4주 내)
4. ✅ **마이페이지 통계 강화** - 사용자 동기부여
5. ✅ **데이터베이스 설계** - 백엔드 개발 준비
6. ✅ **인증 API 구현** - 백엔드 첫 단계

### 중기 (1-2개월 내)
7. ✅ **학습/복습 API 구현** - 핵심 기능 백엔드화
8. ✅ **친구/랭킹 API 구현** - 소셜 기능 백엔드화
9. ✅ **Mock API → 실제 API 전환** - 통합 테스트

### 장기 (2개월 이후)
10. ✅ **푸시 알림** - 사용자 참여도 향상
11. ✅ **실전모드** - 고급 기능

## 💡 개발 전략 제안

### 프론트엔드 우선 개발의 장점
1. **빠른 프로토타이핑**: Mock API로 즉시 UI/UX 검증 가능
2. **명확한 API 스펙**: 프론트엔드 요구사항에 맞춰 백엔드 설계
3. **병렬 개발 가능**: 프론트엔드 완성 후 백엔드 개발 시작

### 다음 단계 액션 아이템
1. **지금 바로**: 홈 화면 추천 자격증 섹션 구현 시작
2. **이번 주**: 친구 기능 기본 UI 완성
3. **다음 주**: 마이페이지 통계 추가

## 🎯 핵심 원칙

1. **사용자 경험 우선**: 핵심 기능부터 완성도 높게 구현
2. **점진적 개선**: MVP → 확장 기능 순서로 개발
3. **기술 부채 최소화**: Mock API 구조를 실제 API와 유사하게 설계
4. **테스트 가능성**: 각 기능을 독립적으로 테스트 가능하도록 구성
